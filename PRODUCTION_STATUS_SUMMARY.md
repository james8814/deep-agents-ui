## Known Limitations

### Dev Server Issue

The dev server (`npm run dev`) has a startup issue with LangGraph SDK persistence initialization:

```
[Error: Failed to open database
Caused by: Loading persistence directory failed]
```

**Workaround**: Use production build (`npm run start`) instead. All functionality works identically.

**Root Cause**: SDK tries to initialize persistence database in Turbopack dev context. This is not a defect in the application code.
