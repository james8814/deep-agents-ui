### RS-001: 筛选器数据过滤逻辑缺失

**优先级**: P1 (高)
**影响**: 用户体验受损，不符合设计基准
**位置**: `src/app/components/WorkPanelV527.tsx:193-206`

#### 设计基准 (Section 5.3.4)

根据设计基准，工作日志列表应该根据筛选器选择进行过滤：

```
筛选器方案规格:
├── 选中"全部"时：显示所有任务的日志 (12 条)
├── 选中"分析竞品"时：仅显示该任务的日志 (3 条)
├── 选中"撰写 PRD"时：仅显示该任务的日志 (2 条)
└── 选中"检查代码"时：仅显示该任务的日志 (7 条)
```

#### 当前实现

```typescript
// WorkPanelV527.tsx:193-206
<div className="space-y-2 p-3">
  {tasksWithIndex.map((task) => (
    <StepGroup
      key={task.id}
      taskId={task.id}
      taskContent={task.content}
      status={task.status}
      logs={logsByTaskId[task.id] || []}  // ❌ 显示所有任务的日志
      collapsed={isCollapsed(task.id, task.status === "in_progress")}
      highlighted={selectedTaskId === task.id}  // ✅ 仅高亮选中任务
      onToggleCollapse={handleToggleCollapse(task.id)}
    />
  ))}
</div>
```

**问题**: 所有任务的 `StepGroup` 都会显示，`selectedTaskId` 仅用于高亮，**没有过滤功能**。

#### 正确实现

```typescript
// 应该添加过滤逻辑
const filteredTasks = useMemo(() => {
  if (selectedTaskId === null) {
    // 选中"全部"，显示所有任务
    return tasksWithIndex;
  } else {
    // 选中特定任务，只显示该任务
    return tasksWithIndex.filter((task) => task.id === selectedTaskId);
  }
}, [tasksWithIndex, selectedTaskId]);

// 在渲染时使用 filteredTasks
<div className="space-y-2 p-3">
  {filteredTasks.map((task) => (
    <StepGroup ... />
  ))}
</div>
```

#### 修复方案

**修改文件**: `src/app/components/WorkPanelV527.tsx`

**修改位置**: 在 Step Groups 渲染前添加过滤逻辑（约第 187 行前）

**代码修改**:

```typescript
// 过滤任务列表（根据选中的任务）
```

### RS-002: ESLint 警告 - macOS 元数据文件

**优先级**: P3 (低)
**影响**: 无功能影响，仅构建日志噪音
**位置**: `._*.tsx` 文件

#### 问题描述

```
/Volumes/0-/jameswu projects/langgraph_test/deep-agents-ui/._next-env.d.ts
  1:0  error  Parsing error: Invalid character
```

**原因**: macOS 在外部驱动器上创建 `._` 前缀的元数据文件（存储扩展属性），这些是二进制文件。

#### 解决方案

**选项 A** (推荐): `.gitignore` 中忽略
```
# macOS metadata files
._*
.DS_Store
```

**选项 B**: 清理现有文件
```bash
find . -name "._*" -type f -delete
```

**状态**: ℹ️ 可忽略 - 不影响功能，仅构建日志噪音
