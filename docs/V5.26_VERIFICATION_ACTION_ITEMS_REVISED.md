## P1 - 中优先级 (改进建议)

### Task 4: FileData 类型修正评估

**背景**:
- 当前 files 类型：`Record<string, string>`（完整字符串）
- 设计建议：`Record<string, FileData>`（行数组 + 时间戳）

**FileData 类型定义**:
```typescript
interface FileData {
  content: string[];      // 行数组，支持行级 diff
  created_at?: string;    // 创建时间戳
  modified_at?: string;   // 修改时间戳
}
```

**评估选项**:
- [ ] **选项 A**: 修正为 FileData 类型（支持行级 diff 和时间戳）
- [ ] **选项 B**: 保持现状（简单字符串）

**影响评估**:
- 实施成本：中等（需修改多处 files 使用位置）
- 用户价值：支持更精细的文件变化追踪

**涉及文件**:
- `src/app/types/types.ts` (FileItem 类型)
- `src/app/hooks/useChat.ts` (StateType 定义)
- `src/app/components/ContextPanel.tsx` (Files Tab 渲染)

**下一步**:
- [ ] 技术负责人评估需求
- [ ] 决策是否修正类型
- [ ] 如修正，制定迁移计划

### Task 5: LogEntry.elapsed_time 字段添加

**背景**:
- 当前 LogEntry 类型缺少 elapsed_time 字段
- 用途：显示工具执行耗时（如 "0.5s"）

**类型定义**:
```typescript
type LogEntry = {
  type: "tool_call" | "tool_result";
  tool_name?: string;
  tool_input?: Record<string, unknown>;
  tool_output?: string;
  tool_call_id?: string;
  status?: "success" | "error";
  elapsed_time?: number;  // 新增：毫秒
};
```

**评估选项**:
- [ ] **选项 A**: 添加 elapsed_time 字段
- [ ] **选项 B**: 保持现状

**影响评估**:
- 实施成本：低（只需添加字段）
- 用户价值：了解工具执行性能

**涉及文件**:
- `src/app/types/types.ts` (LogEntry 类型)
- `pmagent/src/backends/state_tracking.py` (性能指标输出)

**下一步**:
- [ ] 与后端对齐 elapsed_time 是否已收集
- [ ] 如已收集，添加字段到类型定义
- [ ] 在 UI 中显示耗时信息
