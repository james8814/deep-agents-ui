### 埋点实现模式

所有埋点使用统一的条件调用模式：

```typescript
if (typeof window !== "undefined" && (window as any).gtag) {
  (window as any).gtag("event", "event_name", {
    // 事件参数
  });
}
```

**特点**:

- 所有调用都有防御性检查（`window` 存在 + `gtag` 函数存在）
- 如果未注入 gtag 脚本，所有埋点静默失败，不影响功能
- 未发现 gtag 全局脚本注入代码（可能在部署配置或 CDN 层）

## 3. 错误追踪

| 工具                         | 集成方式   | 覆盖范围                |
| ---------------------------- | ---------- | ----------------------- |
| React ErrorBoundary          | 自定义组件 | 全局 React 渲染错误捕获 |
| console.error / console.warn | 原生控制台 | 分散在 30+ 文件中       |

**ErrorBoundary 详情** (`src/app/components/ErrorBoundary.tsx`):

- React class component 实现
- 支持 3 个级别：page、section、inline
- 提供 `onError` 回调（但未连接到外部错误追踪服务）
- 支持错误重置和重试

**缺失项**:

- 无外部错误追踪服务（Sentry、Bugsnag 等）
- 无前端性能监控（Web Vitals、Real User Monitoring）
- 无结构化日志收集（console 输出仅在浏览器 DevTools 可见）

## 4. 埋点覆盖评估

| 业务场景      | 是否有埋点 | 建议                                   |
| ------------- | ---------- | -------------------------------------- |
| 用户注册/登录 | ❌ 无      | 建议添加注册转化、登录成功率事件       |
| 消息发送      | ✅ 有      | `message_sent` 事件                    |
| 文件上传      | ❌ 无      | 建议添加上传成功/失败/文件类型分布事件 |
| HIL 中断审批  | ❌ 无      | 建议添加审批/拒绝比例事件              |
| 子代理使用    | ❌ 无      | 建议添加子代理触发频率事件             |
| 会话管理      | ❌ 无      | 建议添加会话创建、切换、平均时长事件   |
| 错误发生      | ❌ 无上报  | ErrorBoundary 捕获但未上报到外部服务   |
| 页面性能      | ❌ 无      | 建议添加 Web Vitals 监控               |
