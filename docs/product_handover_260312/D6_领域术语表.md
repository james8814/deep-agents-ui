## 业务术语

| 术语 | 含义 | 上下文 |
|------|------|--------|
| OPDCA | Observe-Plan-Do-Check-Adapt，五阶段工作流循环 | Agent 执行任务的主工作流方法论 |
| HIL / Human-in-the-Loop | 人机交互回路 | Agent 在关键决策点暂停等待用户审批 |
| Interrupt | 中断 | HIL 触发的暂停点，Agent 等待用户输入后继续 |
| Resume | 恢复 | 用户审批后恢复被中断的 Agent 执行 |
| Deliverable | 交付物 | Agent 生成的最终产品文档（PRD、设计文档等） |
| Stream / Streaming | 流式传输 | LangGraph SDK 的实时数据推送机制（WebSocket） |
| Checkpoint | 检查点 | LangGraph 的状态快照，用于中断恢复和历史回溯 |
| Deep Agent | 深度代理 | 基于 LangGraph 的增强型 AI 代理，支持规划、文件操作和子代理委派 |
| Bearer Token | 令牌认证 | JWT 格式的认证令牌，通过 HTTP Authorization 头传递 |
| Fetch Interceptor | 请求拦截器 | 全局 fetch 猴子补丁，自动为 API 请求添加认证头 |
| Design Token | 设计令牌 | CSS 变量形式的设计系统基础值（颜色、间距、圆角等） |
| Feature Flag | 功能标记 | 控制特定功能启用/禁用的开关（如 Ant Design X 组件） |

## 易混淆概念辨析

| 概念 A | 概念 B | 区别 |
|--------|--------|------|
| `Thread` 会话线程 | `Message` 消息 | Thread 是一次完整对话，包含多条 Message |
| `ToolCall` 工具调用 | `SubAgent` 子代理 | ToolCall 是单个工具操作；SubAgent 是独立的子 AI，内部可能执行多个 ToolCall |
| `files` 工作区文件 | `FileAttachment` 附件 | files 是 Agent 在工作区创建的文件（后端状态）；FileAttachment 是用户上传的附件 |
| `interrupt` 中断 | `error` 错误 | interrupt 是 HIL 设计的正常暂停；error 是执行异常 |
| `access_token` | `auth_token` | 相同概念的不同名称：API 返回用 access_token，存储用 auth_token 键名 |
| `SubAgent.status` | `SubagentStatus` | 前者是数据模型状态（pending/active/completed/error）；后者是 UI 显示状态（pending/running/success/error） |
| `deploymentUrl` | `AUTH_SERVER` / `API_SERVER` | deploymentUrl 是用户配置的 LangGraph 服务地址；AUTH_SERVER/API_SERVER 是环境变量定义的后端地址 |
