## 3. 安全风险项

| 风险 | 位置 | 严重程度 | 说明 |
|------|------|---------|------|
| JWT 存储在 localStorage | `src/contexts/AuthContext.tsx`, `src/lib/fetchInterceptor.ts` | 高 | XSS 攻击可窃取 Token。代码注释中已意识到此问题（`src/middleware.ts:143`："比 localStorage 更安全"），但未实施 HttpOnly Cookie 方案 |
| 全局 fetch 猴子补丁 | `src/lib/fetchInterceptor.ts` | 中 | 覆盖 `window.fetch` 可能与第三方库冲突；Token 泄露到非预期 URL 的风险 |
| `as any` 绕过类型检查 | 26 处，集中在生产代码 | 中 | 运行时可能出现未预期的类型错误 |
| Token 过期 30 秒缓冲 | `src/contexts/AuthContext.tsx` | 低 | 客户端 JWT 解码可被篡改（不影响安全性，因后端验证） |

**安全优势**:
- ✅ 无 `dangerouslySetInnerHTML` 使用
- ✅ 无 `innerHTML` 直接操作
- ✅ Next.js Middleware 级别的路由保护
- ✅ 自动 Token 刷新机制
