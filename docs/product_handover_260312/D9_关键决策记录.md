## 决策记录

### 从 LangSmith API Key 迁移到 Bearer Token 认证

- **可信度**: 高
- **时间**: 2026-03 (commits: `c7cfc6d`, `068faa9`, `577c5de`, `60a9f26`)
- **内容**: 移除了 LangSmith API Key 认证方式，全面转向 JWT Bearer Token 认证
- **证据**:
  - `c7cfc6d` — "Update all components to remove langsmithApiKey references"
  - `068faa9` — "Update page.tsx to remove apiKey prop from ClientProvider"
  - `577c5de` — "refactor(auth): 统一 TOKEN_KEY + 删除死代码 + 简化 Token 注入"
  - `60a9f26` — "统一 Token 注入架构 + 消除代码质量问题"
  - `src/app/page.tsx:282-283` — 残留注释："移除 API Key，使用 Bearer Token 认证"
- **推断原因**: LangSmith API Key 是共享密钥，无法区分用户；Bearer Token 支持多用户隔离和权限控制

### 采用 fetchInterceptor 全局 fetch 猴子补丁

- **可信度**: 高
- **时间**: 2026-03 (commit: `0a10d7b`)
- **内容**: 使用猴子补丁方式覆盖 `window.fetch`，自动为 API 请求添加 Bearer Token
- **证据**:
  - `src/lib/fetchInterceptor.ts` — 完整的 fetch 覆盖实现
  - `src/components/ClientInitializer.tsx` — 在应用初始化时加载 interceptor
  - CLAUDE.md 中记录了此架构决策
- **推断原因**: LangGraph SDK 内部使用 `fetch` 发送请求，无法通过 SDK 配置注入所有请求的认证头；猴子补丁是最低侵入性的方案。同时为 `ClientProvider.tsx` 使用 `onRequest` hook 提供了双重保障

### 选择 localStorage 存储 JWT 而非 HttpOnly Cookie

- **可信度**: 高
- **时间**: 2026-03
- **内容**: JWT Token 存储在 localStorage，而非使用更安全的 HttpOnly Cookie
- **证据**:
  - `src/contexts/AuthContext.tsx` — `localStorage.setItem(TOKEN_KEY, token)`
  - `src/middleware.ts:143` — 注释："比 localStorage 更安全（不易被 XSS 窃取）"
  - CLAUDE.md — 明确记录 "Cookie-based auth NOT used due to cross-port cookie limitations"
- **推断原因**: 前端(:3000)、Auth Server(:8000)、LangGraph Server(:2024) 运行在不同端口，跨端口 Cookie 无法正常工作。团队已意识到安全隐患但因技术限制做出此妥协
