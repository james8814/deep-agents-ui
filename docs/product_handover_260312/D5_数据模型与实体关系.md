# 数据模型与实体关系

> 本项目为纯前端应用，不直接管理数据库。数据模型以 TypeScript 类型/接口形式定义，对应后端（LangGraph Server + Auth Server）返回的数据结构。

## 1. ER 关系图

```
[Thread] 1 ──── * [Message]
    |                |
    |                * ──── * [ToolCall]
    |                |
    |                * ──── * [SubAgent]
    |                         |
    |                         * ──── * [LogEntry]
    |
```

## 2. 实体详情

### Thread（会话线程）

**来源**: `src/app/hooks/useThreads.ts`
**业务含义**: 一次完整的用户与 AI 的对话会话

| 字段        | 类型               | 业务含义      | 约束          | 备注                            |
| ----------- | ------------------ | ------------- | ------------- | ------------------------------- |
| id          | `string`           | 线程唯一标识  | UUID          | 由 LangGraph Server 生成        |
| updatedAt   | `Date`             | 最后更新时间  | -             | 用于排序                        |
| status      | `Thread["status"]` | 线程状态      | 枚举          | idle / busy / interrupted 等    |
| title       | `string`           | 会话标题      | 最长 50 字符  | 取自第一条用户消息前 50 个字符  |
| description | `string`           | 会话描述      | 最长 100 字符 | 取自第一条 AI 回复前 100 个字符 |
| assistantId | `string?`          | 关联的助手 ID | 可选          | -                               |

**关键关系**:

- 属于 → User（通过 JWT Token 关联）
- 拥有多个 → Message（通过 LangGraph 状态管理）

### Message（消息）

**来源**: `@langchain/core` 的 `Message` 类型
**业务含义**: 对话中的单条消息

| 字段              | 类型                      | 业务含义          | 约束              | 备注             |
| ----------------- | ------------------------- | ----------------- | ----------------- | ---------------- |
| id                | `string`                  | 消息唯一标识      | UUID              | LangGraph 生成   |
| type              | `string`                  | 消息类型          | human / ai / tool | 发送方标识       |
| content           | `string \| ContentPart[]` | 消息内容          | -                 | 支持文本和多模态 |
| tool_calls        | `ToolCall[]`              | 工具调用列表      | AI 消息独有       | -                |
| name              | `string?`                 | 工具名称          | tool 消息独有     | -                |
| tool_call_id      | `string?`                 | 对应的工具调用 ID | tool 消息独有     | -                |
| additional_kwargs | `Record<string, any>`     | 附加参数          | 可选              | 包含 attachments |

**关键关系**:

- 属于 → Thread
- 拥有多个 → ToolCall（AI 消息）
- 拥有多个 → FileAttachment（通过 additional_kwargs.attachments）

### ToolCall（工具调用）

**来源**: `src/app/types/types.ts`
**业务含义**: Agent 执行的单个工具操作

| 字段                | 类型                      | 业务含义         | 约束 | 备注                                                  |
| ------------------- | ------------------------- | ---------------- | ---- | ----------------------------------------------------- |
| id                  | `string`                  | 工具调用唯一标识 | -    | -                                                     |
| name                | `string`                  | 工具名称         | -    | search, write_file 等                                 |
| args                | `Record<string, unknown>` | 工具参数         | -    | JSON 格式                                             |
| result              | `string?`                 | 执行结果         | 可选 | 工具返回值                                            |
| status              | `string`                  | 执行状态         | 枚举 | pending/completed/error/interrupted/cancelled/timeout |
| cancellation_reason | `string?`                 | 取消原因         | 可选 | -                                                     |
| timeout_seconds     | `number?`                 | 超时时长         | 可选 | -                                                     |

**关键关系**:

- 属于 → Message（AI 类型的消息）

### SubAgent（子代理）

**来源**: `src/app/types/types.ts`
**业务含义**: 被主 Agent 委派的专业化子代理

| 字段         | 类型                       | 业务含义       | 约束 | 备注                            |
| ------------ | -------------------------- | -------------- | ---- | ------------------------------- |
| id           | `string`                   | 子代理实例 ID  | -    | -                               |
| name         | `string`                   | 子代理显示名称 | -    | -                               |
| subAgentName | `string`                   | 子代理类型标识 | -    | research_agent, design_agent 等 |
| input        | `Record<string, unknown>`  | 输入参数       | -    | 委派任务描述                    |
| output       | `Record<string, unknown>?` | 输出结果       | 可选 | 完成后的返回值                  |
| status       | `string`                   | 执行状态       | 枚举 | pending/active/completed/error  |
| logs         | `LogEntry[]?`              | 执行日志       | 可选 | 工具调用记录                    |

**关键关系**:

- 属于 → Message
- 拥有多个 → LogEntry

### LogEntry（日志条目）

**来源**: `src/app/types/subagent.ts`
**业务含义**: 子代理执行过程中的工具调用日志

| 字段         | 类型                       | 业务含义        | 约束                         | 备注                 |
| ------------ | -------------------------- | --------------- | ---------------------------- | -------------------- |
| type         | `string`                   | 日志类型        | `tool_call` \| `tool_result` | -                    |
| tool_name    | `string?`                  | 工具名称        | 可选                         | -                    |
| tool_input   | `Record<string, unknown>?` | 工具输入参数    | 可选                         | -                    |
| tool_output  | `string?`                  | 工具输出结果    | 可选                         | -                    |
| tool_call_id | `string?`                  | 工具调用关联 ID | 可选                         | 用于配对 call/result |
| status       | `string?`                  | 执行状态        | `success` \| `error`         | -                    |

**来源**: `src/app/types/types.ts`

| 字段      | 类型     | 业务含义     | 约束                                      | 备注 |
| --------- | -------- | ------------ | ----------------------------------------- | ---- |
| id        | `string` | 任务唯一标识 | -                                         | -    |
| content   | `string` | 任务描述内容 | -                                         | -    |
| status    | `string` | 任务状态     | `pending` \| `in_progress` \| `completed` | -    |
| updatedAt | `Date?`  | 最后更新时间 | 可选                                      | -    |

**关键关系**:

- 属于 → Thread（通过 StateType.todos）

### FileMetadata（文件元数据）

**来源**: `src/app/types/types.ts`
**业务含义**: Agent 工作区中的文件信息

| 字段      | 类型     | 业务含义     | 约束       | 备注     |
| --------- | -------- | ------------ | ---------- | -------- |
| path      | `string` | 文件完整路径 | -          | 虚拟路径 |
| name      | `string` | 文件名       | -          | -        |

| addedAt   | `number` | 添加时间     | 毫秒时间戳 | -        |
| size      | `number` | 文件大小     | 字符数     | 非字节数 |

### FileAttachment（文件附件）

**来源**: `src/app/types/types.ts`
**业务含义**: 用户上传并附加到消息的文件引用

## 3. 数据流向

### 核心数据流（Graph State）

```
LangGraph Server (后端)
    ↓ WebSocket stream (StateType)
useStream Hook
    ↓ 解构
useChat Hook
    ├── messages → ChatMessage 组件
    ├── todos → ContextPanel.TasksTab
    ├── files → ContextPanel.FilesTab
    ├── subagent_logs → SubAgentPanelCard
    └── interrupt → ToolApprovalInterrupt
```

### 认证数据流

```
用户输入凭据 → Auth Server (/auth/login-cookie)
    ↓ JWT Response
localStorage["auth_token"] ← 存储
    ↓ fetchInterceptor 读取
所有 API 请求 → Authorization: Bearer {token}
    ↓
Cookie["auth_token"] ← 同步设置
    ↓ Next.js Middleware 读取
路由保护决策
```

### 文件上传数据流

```
用户拖拽文件 → FileUploadZone 验证
    ↓ XHR POST /api/upload
Auth Server 存储文件
    ↓ 返回虚拟路径
FileChip 显示 ← path: "/uploads/{user_id}/{filename}"
    ↓ 发送消息时
message.additional_kwargs.attachments ← [{ path, filename, size }]
    ↓ stream.submit()
LangGraph Agent 通过 FilesystemMiddleware 读取文件
```

## 4. 存储策略

| 策略       | 实现方式                | 涉及实体                         | 代码位置                             |
| ---------- | ----------------------- | -------------------------------- | ------------------------------------ |
| 前端持久化 | localStorage            | Config, Token, Theme, Settings   | `src/lib/config.ts`, `src/contexts/` |
| 会话持久化 | LangGraph Checkpoint    | Thread, Message, TodoItem, Files | 后端管理                             |
| URL 状态   | nuqs (URL query params) | threadId, assistantId, sidebar   | `src/app/page.tsx`                   |
| 文件存储   | Auth Server 文件系统    | FileAttachment                   | `src/api/upload.ts`                  |

> 注意：本前端不直接操作数据库。所有持久化数据通过后端 API（LangGraph Server + Auth Server）间接管理。
