# 用户旅程与核心场景

## 1. 普通用户核心旅程

### 场景: 首次使用 — 注册登录并开始对话

**用户目标**: 注册账号，配置系统，并开始与 AI 助手对话
**前置条件**: 系统已部署（Auth Server + LangGraph Server + 前端）

| 步骤 | 用户操作 | 页面/组件 | 调用 API | 系统响应 |
|------|---------|----------|---------|---------|
| 1 | 访问首页 | `/` | - | Middleware 检测无 Token，重定向到 `/login` |
| 2 | 点击"注册" | `/login` → `/register` | - | 跳转注册页 |
| 3 | 填写用户名、邮箱、密码 | `/register` RegisterForm | - | 前端表单验证 |
| 4 | 点击注册 | `/register` | `POST /auth/register` | 注册成功，提示跳转登录 |
| 5 | 填写用户名、密码 | `/login` LoginForm | - | 前端表单验证 |
| 6 | 点击登录 | `/login` | `POST /auth/login-cookie` | JWT Token 存入 localStorage，跳转主页 |
| 7 | 首次进入 | ConfigDialog | - | 显示配置对话框：输入 deploymentUrl 和 assistantId |
| 8 | 填写配置并确认 | ConfigDialog | - | 配置存入 localStorage，连接 LangGraph Server |
| 9 | 在输入框输入需求 | ChatInterface / InputArea | - | 输入框支持 Shift+Enter 换行 |
| 10 | 点击发送 | InputArea | `stream.submit()` | Agent 开始处理，流式显示回复 |

**分支路径**:
- 如果注册失败: → 显示错误信息（用户名已存在等）
- 如果 LangGraph Server 不可达: → fetch 错误，无法发送消息

来源: `src/app/(auth)/login/page.tsx`, `src/app/(auth)/register/page.tsx`, `src/app/components/ConfigDialog.tsx`

### 场景: 日常对话 — 发送消息并查看 Agent 工作过程

**用户目标**: 与 AI 助手进行产品规划对话
**前置条件**: 已登录，已配置

| 步骤 | 用户操作 | 页面/组件 | 调用 API | 系统响应 |
|------|---------|----------|---------|---------|
| 1 | 输入产品需求描述 | InputArea | - | 文本编辑 |
| 2 | 点击发送 | InputArea | `stream.submit()` | 显示用户消息，AI 开始回复 |
| 3 | 观察 AI 回复 | ChatMessage | WebSocket stream | 逐字流式渲染 Markdown 内容 |
| 4 | 查看工具调用 | ToolCallBox | - | 展开/折叠工具调用详情（参数+结果） |
| 5 | 查看子代理工作 | SubAgentIndicator | - | 展开查看子代理思考链和工具日志 |
| 6 | 查看 OPDCA 阶段 | OPDCAStageDisplay | - | 时间线显示当前处于哪个阶段 |
| 7 | 查看任务列表 | ContextPanel → TasksTab | - | 右侧栏显示 Agent 规划的任务列表 |
| 8 | 查看生成的文件 | ContextPanel → FilesTab | - | 右侧栏显示工作区文件列表 |
| 9 | 查看交付物 | DeliveryCard | - | 卡片形式展示 PRD、设计文档等 |

**分支路径**:
- 如果需要重新生成: → 点击重新生成按钮，调用 `regenerateLastMessage()`
- 如果需要停止: → 点击停止按钮，调用 `stopStream()`

来源: `src/app/hooks/useChat.ts`, `src/app/components/ChatInterface.tsx`

### 场景: 上传文件并附加到消息

**用户目标**: 向 AI 提供文件参考资料（需求文档、设计稿等）
**前置条件**: 已登录

| 步骤 | 用户操作 | 页面/组件 | 调用 API | 系统响应 |
|------|---------|----------|---------|---------|
| 1 | 拖拽文件到上传区域 / 点击选择 | FileUploadZone | - | 显示上传区域高亮 |
| 2 | 等待上传 | FileUploadZone | `POST /api/upload` (XHR) | 显示上传进度条 |
| 3 | 上传完成 | FileChip | - | 显示文件预览芯片（文件名+大小+删除按钮） |
| 4 | 输入文本消息 | InputArea | - | 可同时附加多个文件 |
| 5 | 发送消息 | InputArea | `stream.submit()` | 文件路径作为附件发送给 Agent |

**分支路径**:
- 如果文件类型不支持: → 显示类型错误提示
- 如果文件超过 100MB: → 显示大小限制提示
- 如果上传失败: → Promise reject，可重试

来源: `src/app/components/FileUploadZone.tsx`, `src/api/upload.ts`

### 场景: HIL 中断审批 — 审批 Agent 的关键决策

**用户目标**: 审核 Agent 的工具调用请求，决定是否批准
**前置条件**: Agent 执行到 HIL 检查点

| 步骤 | 用户操作 | 页面/组件 | 调用 API | 系统响应 |
|------|---------|----------|---------|---------|
| 1 | 收到中断通知 | 浏览器通知（如果标签页失焦） | - | 浏览器 Notification API |
| 2 | 查看中断详情 | ToolCallBox（自动展开） | - | 显示被中断的工具调用和参数 |
| 3 | 查看审批表单 | ToolApprovalInterrupt | - | 显示批准/拒绝按钮 |
| 4a | 点击批准 | ToolApprovalInterrupt | `Command({ resume })` | Agent 继续执行 |
| 4b | 点击拒绝 | ToolApprovalInterrupt | `Command({ resume })` | Agent 处理拒绝逻辑 |

**分支路径**:
- 如果用户长时间不操作: → Agent 保持暂停，通过 Checkpoint 持久化状态
- 如果用户切换到其他会话再回来: → 中断状态仍在，可继续审批

来源: `src/app/hooks/useInterruptNotification.ts`, `src/app/components/ToolApprovalInterrupt.tsx`

### 场景: 会话管理 — 切换和浏览历史会话

**用户目标**: 查看历史对话，继续之前的工作
**前置条件**: 已有历史会话

| 步骤 | 用户操作 | 页面/组件 | 调用 API | 系统响应 |
|------|---------|----------|---------|---------|
| 1 | 查看左侧会话列表 | ThreadList | `client.threads.search()` | 分页加载会话列表 |
| 2 | 滚动加载更多 | ThreadList | 下一页请求 | SWR 无限分页 |
| 3 | 按状态筛选 | ThreadList 筛选按钮 | 条件搜索 | 过滤显示 interrupted 等状态 |
| 4 | 点击目标会话 | ThreadList → ThreadItem | - | 更新 URL `?threadId=xxx` |
| 5 | 查看历史消息 | ChatInterface | 加载 thread 状态 | 渲染完整对话历史 |
| 6 | 新建会话 | ThreadList → 新建按钮 | - | 清除 threadId，开始新对话 |

来源: `src/app/hooks/useThreads.ts`, `src/app/components/ThreadList.tsx`

## 3. 异常与边界体验

| 场景 | 用户看到什么 | 实现方式 | 代码位置 |
|------|------------|---------|---------|
| 无权限访问 | 重定向到登录页 | Next.js Middleware Cookie 检查 | `src/middleware.ts` |
| 网络错误 | 流式错误被静默处理；上传失败显示错误信息 | try-catch + console.warn | `src/app/hooks/useChat.ts`, `src/api/upload.ts` |
| 组件崩溃 | ErrorBoundary 显示错误卡片，支持重试 | React ErrorBoundary（class component） | `src/app/components/ErrorBoundary.tsx` |
| Token 过期 | 自动刷新；刷新失败则清除登录态，重定向到登录页 | fetchWithCredentials 401 拦截 | `src/api/client.ts` |
| LangGraph Server 不可达 | fetch 错误，无法发送消息 | handleStreamError | `src/app/hooks/useChat.ts` |
| Schema 版本不兼容 | 显示 SchemaVersionWarning 警告 | 版本比较组件 | `src/app/components/SchemaVersionWarning.tsx` |
| 文件类型不支持 | 上传被拒绝，显示类型限制提示 | isAllowedFileType 检查 | `src/api/upload.ts` |
| 数据为空 | 会话列表为空时显示"无会话"提示 | 条件渲染 | `src/app/components/ThreadList.tsx` |
