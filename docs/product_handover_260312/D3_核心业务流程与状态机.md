# 核心业务流程与状态机

## 1. 核心业务流程

### 流程: 用户认证

**触发条件**: 用户访问受保护页面且无有效 Token
**参与角色**: 用户, Auth Server

```
访问受保护页面 → Middleware 检查 Cookie → 无 Token → 重定向 /login
    → 用户填写表单 → 提交登录 → Auth Server 验证
        → 成功 → 存储 JWT → 设置 Cookie → 跳转主页
        → 失败 → 显示错误信息
```

**步骤详情**:

| 步骤 | 操作 | 系统行为 | 代码位置 |
|------|------|---------|---------|
| 1 | 访问 `/` | Middleware 检查 `auth_token` Cookie | `src/middleware.ts:36-52` |
| 2 | 无 Token | 重定向到 `/login` | `src/middleware.ts:44` |
| 3 | 填写用户名密码 | 前端表单验证 | `src/app/(auth)/login/page.tsx` |
| 4 | 点击登录 | 调用 `AuthContext.login()` | `src/contexts/AuthContext.tsx` |
| 5 | 发送请求 | `POST /auth/login-cookie` | `src/api/auth.ts` |
| 6 | 成功 | 存储 Token 到 localStorage + 设置 Cookie | `src/contexts/AuthContext.tsx` |
| 7 | 自动跳转 | `router.push("/")` | `src/app/(auth)/login/page.tsx` |

### 流程: 发送消息与 Agent 对话

**触发条件**: 用户在输入框输入文本并点击发送
**参与角色**: 用户, LangGraph Server, AI Agent

```
用户输入 → 发送消息 → stream.submit() → LangGraph WebSocket
    → Agent 流式回复 → 实时渲染消息
        ↓ (如有工具调用)
    → 显示 ToolCallBox → 工具执行结果
        ↓ (如有子代理)
    → 显示 SubAgentIndicator → 子代理日志
        ↓ (如触发 HIL)
    → 中断等待 → 用户审批 → 恢复执行
```

**步骤详情**:

| 步骤 | 操作 | 系统行为 | 代码位置 |
|------|------|---------|---------|
| 1 | 输入文本 | 更新 InputArea 状态 | `src/app/components/InputArea.tsx` |
| 2 | 附加文件(可选) | 上传到 Auth Server，获取虚拟路径 | `src/api/upload.ts` |
| 3 | 点击发送 | 调用 `sendMessage(content, attachments)` | `src/app/hooks/useChat.ts` |
| 4 | 构建消息 | 创建 HumanMessage，附加 attachments 到 additional_kwargs | `src/app/hooks/useChat.ts` |
| 5 | 提交流 | `stream.submit({ messages }, config)` | `src/app/hooks/useChat.ts` |
| 6 | 流式接收 | WebSocket 持续推送 StateType 更新 | `@langchain/langgraph-sdk` |
| 7 | 渲染消息 | ChatMessage 组件实时更新 | `src/app/components/ChatMessage.tsx` |
| 8 | 渲染工具调用 | ToolCallBox 显示参数和结果 | `src/app/components/ToolCallBox.tsx` |
| 9 | 渲染子代理 | SubAgentIndicator 显示状态 | `src/app/components/SubAgentIndicator.tsx` |

### 流程: Human-in-the-Loop 中断审批

**触发条件**: Agent 执行到需要用户确认的关键决策点
**参与角色**: 用户, AI Agent

```
Agent 触发 interrupt → stream.interrupt 有值 → 显示中断 UI
    → 用户查看中断详情 → 批准/拒绝
        → 批准 → resumeInterrupt(value) → Command({ resume }) → Agent 继续
        → 拒绝 → resumeInterrupt(rejection) → Agent 处理拒绝
```

**步骤详情**:

| 步骤 | 操作 | 系统行为 | 代码位置 |
|------|------|---------|---------|
| 1 | Agent 中断 | `stream.interrupt` 变为非空 | `src/app/hooks/useChat.ts` |
| 2 | 显示中断 Banner | ToolCallBox 自动展开中断的工具调用 | `src/app/components/ToolCallBox.tsx` |
| 3 | 通知提醒 | 标签页失焦时发送浏览器通知 | `src/app/hooks/useInterruptNotification.ts` |
| 4 | 用户审批 | 点击批准/拒绝按钮 | `src/app/components/ToolApprovalInterrupt.tsx` |
| 5 | 恢复执行 | `stream.submit(null, { command: { resume } })` | `src/app/hooks/useChat.ts:resumeInterrupt` |

### 流程: 文件上传与消息附加

**触发条件**: 用户拖拽或选择文件到上传区域
**参与角色**: 用户, Auth Server

```
用户选择/拖拽文件 → 类型验证 → 大小检查 (≤100MB)
    → XHR 上传 → 进度显示 → 获取虚拟路径
    → 显示 FileChip → 用户发送消息时作为附件
```

**步骤详情**:

| 步骤 | 操作 | 系统行为 | 代码位置 |
|------|------|---------|---------|
| 1 | 拖拽/选择文件 | 验证文件类型和大小 | `src/app/components/FileUploadZone.tsx` |
| 2 | 开始上传 | XHR POST `/api/upload` + Bearer Token | `src/api/upload.ts:uploadFile` |
| 3 | 进度更新 | XHR progress 事件回调 | `src/api/upload.ts` |
| 4 | 上传完成 | 返回 `{ path: "/uploads/{user_id}/{filename}" }` | `src/api/upload.ts` |
| 5 | 显示预览 | FileChip 组件显示文件名+删除按钮 | `src/app/components/FileChip.tsx` |
| 6 | 发送消息 | 附件路径加入 `message.additional_kwargs.attachments` | `src/app/hooks/useChat.ts` |

### 流程: 会话切换


| 步骤 | 操作 | 系统行为 | 代码位置 |

## 2. 状态机

### ToolCall 状态机

**状态枚举** (来源: `src/app/types/types.ts`):

| 状态值 | 中文含义 | 说明 |
|--------|---------|------|
| `pending` | 待执行 | 工具调用已创建，等待执行 |
| `completed` / `ok` | 已完成 | 执行成功，有结果 |
| `error` | 错误 | 执行失败 |
| `interrupted` | 已中断 | 被 HIL 暂停，等待用户审批 |
| `cancelled` | 已取消 | 用户取消了执行 |
| `timeout` | 超时 | 执行超时 |

**状态流转**:

```
pending --[执行开始]--> completed/error/timeout
pending --[HIL 触发]--> interrupted
interrupted --[用户批准]--> completed
interrupted --[用户拒绝]--> cancelled
```

| 当前状态 | 触发动作 | 目标状态 | 执行逻辑 | 代码位置 |
|---------|---------|---------|---------|---------|
| pending | Agent 执行工具 | completed | 工具成功返回结果 | `src/app/components/ToolCallBox.tsx` |
| pending | 执行异常 | error | 捕获异常，显示错误信息 | `src/app/components/ToolCallBox.tsx` |
| pending | HIL 配置触发 | interrupted | Agent 在 interruptBefore 暂停 | `src/app/hooks/useChat.ts` |
| interrupted | 用户点击批准 | completed | `resumeInterrupt(approvedValue)` | `src/app/components/ToolApprovalInterrupt.tsx` |
| interrupted | 用户点击拒绝 | cancelled | `resumeInterrupt(rejectionValue)` | `src/app/components/ToolApprovalInterrupt.tsx` |

### SubAgent 状态机

**状态枚举** (来源: `src/app/types/types.ts`, `src/app/types/subagent.ts`):

| 状态值 | 中文含义 | 说明 |
|--------|---------|------|
| `pending` | 等待中 | 子代理已分配，未开始 |
| `active` / `running` | 执行中 | 子代理正在工作 |
| `completed` / `success` | 已完成 | 子代理成功完成任务 |
| `error` | 错误 | 子代理执行出错 |

**状态流转**:

```
pending --[开始执行]--> active
active --[执行成功]--> completed
active --[执行失败]--> error
```

### TodoItem 状态机

**状态枚举** (来源: `src/app/types/types.ts`):

| 状态值 | 中文含义 | 说明 |
|--------|---------|------|
| `pending` | 待处理 | 任务已创建，未开始 |
| `in_progress` | 进行中 | Agent 正在处理该任务 |
| `completed` | 已完成 | 任务完成 |

**状态流转**:

```
pending --[Agent 开始]--> in_progress --[Agent 完成]--> completed
```

## 3. 业务规则清单

| 规则 | 描述 | 约束条件 | 代码位置 |
|------|------|---------|---------|
| Token 认证 | 所有受保护 API 请求必须携带 Bearer Token | 无 Token 返回 401 | `src/lib/fetchInterceptor.ts` |
| Token 自动刷新 | 401 响应时自动尝试刷新 Token（最多 3 次） | 非 `/auth/` 端点 | `src/api/client.ts` |
| 文件大小限制 | 上传文件不超过 100MB | `MAX_UPLOAD_SIZE = 100 * 1024 * 1024` | `src/api/upload.ts` |
| 文件类型限制 | 仅允许 40+ 指定 MIME 类型 | image/*, pdf, doc, xls 等 | `src/api/upload.ts:ACCEPTED_FILE_TYPES` |
| 首次配置 | 无 config 时显示配置对话框 | 检查 localStorage `deep-agent-config-v2` | `src/app/components/ConfigDialog.tsx` |
| 开发模式自动登录 | `NODE_ENV=development` 时自动以 demo 凭据登录 | 仅开发环境 | `src/contexts/AuthContext.tsx` |
| Token 过期检查 | JWT 过期前 30 秒视为过期 | 客户端解码 JWT payload | `src/contexts/AuthContext.tsx` |

## 4. 异常路径处理

| 场景 | 触发条件 | 处理方式 | 代码位置 |
|------|---------|---------|---------|
| 网络断开 | fetch 失败 | console.warn 记录，不显示 UI 错误 | `src/app/hooks/useChat.ts` |
| Token 过期 | API 返回 401 | 自动刷新 Token，失败则清除登录态 | `src/api/client.ts` |
| 上传失败 | 文件上传 XHR 出错 | Promise reject，由组件处理 | `src/api/upload.ts` |
| 流式错误 | WebSocket 中断 | handleStreamError 分类处理 | `src/app/hooks/useChat.ts` |
| 组件崩溃 | React 运行时错误 | ErrorBoundary 捕获，显示错误卡片 | `src/app/components/ErrorBoundary.tsx` |
| Schema 不兼容 | 后端状态格式变更 | SchemaVersionWarning 组件警告 | `src/app/components/SchemaVersionWarning.tsx` |
