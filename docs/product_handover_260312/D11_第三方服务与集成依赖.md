# 第三方服务与集成依赖

## 服务清单

| 服务                    | 用途                    | 集成方式        | 依赖包                     | 配置位置                     | 费用相关                  |
| ----------------------- | ----------------------- | --------------- | -------------------------- | ---------------------------- | ------------------------- |
| LangGraph Server        | AI Agent 执行与流式通信 | SDK (WebSocket) | `@langchain/langgraph-sdk` | `src/lib/config.ts`, `.env`  | 按量（取决于 LLM 提供商） |
| Auth Server (FastAPI)   | 用户认证与文件上传      | REST API        | `src/api/client.ts`        | `.env: NEXT_PUBLIC_AUTH_URL` | 自部署（免费）            |
| Google Analytics (gtag) | 用户行为追踪            | Script 注入     | 无额外依赖                 | HTML head / 组件内           | 免费（标准版）            |

## 集成详情

### LangGraph Server

- **用途**: 运行 AI Agent（deepagents 框架），提供流式消息推送、线程管理、检查点状态
- **集成方式**: `@langchain/langgraph-sdk` SDK（WebSocket + REST）
- **关键代码**:
  - `src/providers/ClientProvider.tsx` — 创建 SDK Client 实例
  - `src/app/hooks/useChat.ts` — 使用 `useStream` 进行流式通信
  - `src/app/hooks/useThreads.ts` — 使用 `client.threads.search()` 查询线程
- **配置项**:
  - `NEXT_PUBLIC_API_URL` 环境变量（默认 `http://localhost:2024`）
  - `deploymentUrl` — 运行时配置（`src/lib/config.ts`，localStorage）
  - `assistantId` — 指定使用的 Agent/Graph ID（默认 `pmagent`）
- **替换难度**: 高
- **替换难度说明**: 整个应用的核心通信依赖于 LangGraph SDK 的 `useStream` 和 `Client` API。替换需要重新实现流式通信、状态管理和线程管理逻辑

### Auth Server (FastAPI)

- **用途**: 用户注册/登录/登出、JWT Token 管理、文件上传存储
- **集成方式**: REST API（`fetch` + `XMLHttpRequest`）
- **关键代码**:
  - `src/api/auth.ts` — 认证 API 调用（register, login, logout, getUserInfo）
  - `src/api/upload.ts` — 文件上传（`POST /api/upload`）
  - `src/api/client.ts` — HTTP 客户端基础层（错误处理、Token 刷新）
  - `src/lib/fetchInterceptor.ts` — 全局 Bearer Token 注入
- **配置项**:
  - `NEXT_PUBLIC_AUTH_URL` 环境变量（默认 `http://localhost:8000`）
- **替换难度**: 中
- **替换难度说明**: Auth API 接口相对标准（JWT 流程），可替换为其他认证服务（Auth0, Supabase Auth 等），但需调整 fetchInterceptor 和 Token 管理逻辑

### Google Analytics (gtag)

- **用途**: 追踪用户交互事件（消息发送、代码复制、工具展开等）
- **集成方式**: `window.gtag()` 全局函数调用（条件检查：`typeof window !== "undefined" && (window as any).gtag`）
- **关键代码**:
  - `src/app/components/ChatInterface.tsx:260-261` — `message_sent` 事件
  - `src/app/components/MessageListEnhanced.tsx:41-42,162-163` — `code_block_copied`, `message_expanded` 事件
  - `src/app/components/InputArea.tsx:147-148` — `input_expand_toggled` 事件
  - `src/app/components/ToolCallBoxEnhanced.tsx:184-185,199-200` — `tool_call_expanded`, `tool_args_copied` 事件
- **配置项**: 需要在 HTML head 中注入 gtag 脚本（当前代码中未发现全局注入脚本，仅有事件调用）
- **替换难度**: 低
- **替换难度说明**: 所有 analytics 调用都是可选的（有条件检查），且采用统一的 `gtag("event", ...)` 模式，可轻松替换为 Mixpanel、PostHog 等
