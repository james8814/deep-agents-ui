## 🤖 多 Agent 测试分工

### Agent 1: UI 结构测试专家
**职责**:
- 核查 Tab 结构、Mini Status Bar、Progress Header
- 使用 Chrome DevTools 获取 DOM 快照
- 对照设计原稿 HTML 行号进行验证

**工具**:
- `mcp__chrome-devtools__take_snapshot`
- `mcp__chrome-devtools__navigate_page`
- Grep (源代码搜索)

**输出**: `docs/test_reports/UI_STRUCTURE_REPORT.md`

---

### Agent 2: Tasks Tab 测试专家
**职责**:
- 核查 Tasks Tab 所有功能
- 执行交互测试（点击、选择、筛选）
- 验证任务状态和进度条

**工具**:
- `mcp__chrome-devtools__click`
- `mcp__chrome-devtools__take_snapshot`
- `mcp__chrome-devtools__fill`

**输出**: `docs/test_reports/TABS_TASKS_REPORT.md`

---

### Agent 3: Files & SubAgents 测试专家
**职责**:
- 核查 Files Tab 和 SubAgents Tab 功能
- 验证文件操作和子代理日志
- 执行端到端功能测试

**工具**:
- `mcp__chrome-devtools__click`
- `mcp__chrome-devtools__wait_for`
- `mcp__chrome-devtools__take_snapshot`

**输出**: `docs/test_reports/TABS_FILES_SUBAGENTS_REPORT.md`

---

### Agent 4: 类型定义与视觉测试专家
**职责**:
- 核查 TypeScript 类型定义
- 验证 CSS 变量和设计系统
- 执行视觉回归测试

**工具**:
- Read (源代码读取)
- Grep (类型搜索)
- `mcp__chrome-devtools__take_screenshot`

**输出**: `docs/test_reports/TYPES_VISUAL_REPORT.md`

## 📊 测试执行流程

### Phase 1: 环境准备 (10 分钟)
```bash
# 1. 确认前端服务运行
curl http://localhost:3000

# 2. 确认有测试数据（执行一个实际任务）
# 如果无任务，创建一个测试任务

# 3. 打开浏览器，导航到测试页面
# http://localhost:3000/?assistantId=pmagent&sidebar=1&context=1
```

### Phase 2: 并行测试执行 (60 分钟)
- Agent 1: UI 结构测试
- Agent 2: Tasks Tab 测试
- Agent 3: Files & SubAgents 测试
- Agent 4: 类型定义与视觉测试

### Phase 3: 报告汇总 (20 分钟)
- 收集所有 Agent 的报告
- 生成综合测试报告
- 计算加权评分
- 生成待办清单

### Phase 4: 审查与决策 (后续)
- 产品负责人审查
- 决策会议
- 修正实施
